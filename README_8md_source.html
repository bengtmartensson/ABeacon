1 # ABeacon
2 
3 This project consists of a class implementing an AMX-compatible beacon
4 on an Arduino with Ethernet card using W5100, W5200, or W5500 chips.
5 If using an Ethernet shield using W5500 or W5200 chip,
6 be sure to use version 2.0.0 (or later) of the Ethernet
7 library, since it supports both chips, auto-detecting during runtime.
8 
9 The AMX beacon protocol is described e.g. [here](http://www.globalcache.com/files/docs/API-GC-100.pdf).
10 
11 This is a singleton class, with no public constructor. Instead, the
12 static "factory" function setup() should be called.
13 The user then has to call the send() function on
14 a regular basis, e.g. every 10 to 60 seconds.
15 Alternatively, checkSend() can be called arbitrarily often,
16 without flooding the LAN. See example/Beacon.ino for an example.
17 
18 The class requires that the Ethernet has been setup before calling newInstance.
19 It then instantiates its own private EthernetUDP instance.
20 
21 The beacon should transmit an UDP broadcast message to IP-address
22 239.255.250.250, port 9131. However, the Arduino library does not
23 consider this IP address to be a broadcast address. Therefore, this
24 implementation instead sends to broadcast address 255.255.255.255.
